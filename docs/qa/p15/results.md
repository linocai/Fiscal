# P15 月度现金流批量编辑日期修复验收

日期：2026-07-17

## 根因与修复

- 旧批量更新先按月份偏移计算日期，随后公共字段赋值又把所有目标覆盖为当前期的绝对日期。
- Build 8 编辑已有系列时还会伪造“当前期后 6 个月”的结束日期，导致后续期次被静默取消。
- 后端现按现有期次逐个保留月份偏移，普通“本次及以后”编辑不再使用客户端推断的结束日期改变系列长度。
- iOS/macOS 编辑页不再为已有系列发送伪结束日期，并明确说明日期会按每月间隔顺延。
- 方向变化会清理不兼容的分类或转入账户引用。
